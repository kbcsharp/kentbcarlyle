#8 Navigating the navigation...

09 September 2018


I want to do a quick post detailing the simple, responsive navigation bar used in my personal website here. I copied the applicable pieces in the codepen below. I am using fontawesome for the github and hamburger menu icon. You can add the link to the codepen CDN in your settings so you don't have to place it in your HTML everytime.


See the Pen Navigating the navigation by Kent Carlyle II (@kbcsharp) on CodePen.


HTML

The HTML is really straightforward here. I have a wrapper around everything. Inside of this I have a header, then a nav elment wrapping a span, heading, and ul with list items. The span has an id of navbar-toggle and this is what contains the hamburger icon. I gave the heading an id of left-nav and the unordered list an id of right-nav.


CSS

The important pieces of the CSS here are the header with display: flex and position: relative. This lets me position absolute the left-nav, right-nav, and navbar-toggle while keeping them from continuing to stretch infinitely on for instance a really wide monitor. If the view width of the page is below 767px then the right-nav is set at display:none and the navbar-toggle is position absolute. This will show the hamburger icon as default on smaller view widths. I also have the right-nav with flex-direction: column, so when the hamburger icon is clicked, the menu items will be vertical. In the single media query, this is changed to row for view widths above 767px. Also, when the view width is greater than 767px, I set the navbar-toggle to display:none to hide the hamburger icon. One last note is that I have a margin-top of 70px on the #right-nav on smaller view widths so it doesn't interfere with the left-nav heading, which in this case is my name. With the larger screen media query, I change the margin to 0.


JavaScript

First, I am creating element objects with variable names rightNav and navbarToggle. rightNav is grabbing the unordered list menu items and navbarToggle is grabbing the hamburger menu icon. Then, I use navbarToggle and add the EventTarget method addEventListener(). This sets up a function that will be called when a specific event occurs -- in this instance it is when the icon is clicked. Since I created an object, I can now use the 'this' keyword which will reference the object in which it is called. So it will reference navbarToggle. Next, it is just using an if statement. If the navbarToggle has a class of active, I want to hide the rightNav menu items, then remove the active class from navBarToggle. This will make it so on the next click, since the active class is gone, the else statement will trigger. This will display the rightNav and then add the active class to navbarToggle.


Conclusion

I'm sure others have done a nav bar just like this or possibly in a much different way. This is simple and worked for me. This brings me back to bootstrap. I am not a big fan of bootstrap but I was thinking maybe it is a really easy way to get a quick responsive navigation bar. However, when I use it on my mac and click the menu on a small screen, I can't get the menu to disappear when clicking again. It seems so difficult to get what you want with bootstrap. I don't understand the love affair with it.
